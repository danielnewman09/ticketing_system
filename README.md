# Ticketing System

Requirements traceability and semantic search dashboard backed by a Neo4j knowledge graph.

## Quick Start

```bash
# 1. Start the Neo4j knowledge graph (Docker required)
codegraph-db start

# 2. Index the codebase into the graph
python -m doxygen_index.cli project . --format neo4j --neo4j-password ticketing-system

# 3. Launch the dashboard
ticketing-dashboard --show
```

Then open **http://localhost:8082**.

Stop the database when done:

```bash
codegraph-db stop
```

## Architecture

```
┌──────────────────────────────────────────────────┐
│  ticketing-dashboard (NiceGUI :8082)             │
│  ┌──────────────┐   ┌──────────────────────────┐ │
│  │ frontend_     │   │ backend_migrated          │ │
│  │ migrated/     │──▶│  • neomodel ORM models    │ │
│  │  • pages/     │   │  • agents/ (LLM tools)    │ │
│  │  • theme.py   │   │  • tools/                 │ │
│  │  • widgets.py │   │  • codebase/indexing      │ │
│  └──────────────┘   └─────────┬────────────────┘ │
│                               │                   │
│                    codegraph.config               │
│                    codegraph.connection           │
└───────────────────────────────┼───────────────────┘
                                │ bolt://localhost:7687
                    ┌───────────▼───────────────────┐
                    │  neo4j-ticketing_system        │
                    │  (Docker: neo4j:5-community)   │
                    │  data/ → codegraph/neo4j/data/ │
                    │  password: ticketing-system    │
                    └───────────────────────────────┘
```

- **Neo4j container** managed by `codegraph-db` — one container per project, named `neo4j-<project_name>`, with data persisted at `codegraph/neo4j/data/`.
- **codegraph** indexes the Python source into typed graph nodes (classes, methods, functions, namespaces, files).
- **backend_migrated** uses `neomodel` ORM models to read/write requirements, components, dependencies, and project metadata.
- **frontend_migrated** is the NiceGUI dashboard with pages for requirements, components, ontology, and dependency review.

## CLI Reference

### Database Management

All commands use the project's `.doxygen-index.toml` for configuration.

```bash
codegraph-db start     # Start Neo4j (creates if needed)
codegraph-db stop      # Stop (data preserved)
codegraph-db status    # Show container status
codegraph-db shell     # Interactive Cypher shell
codegraph-db browser   # Open Neo4j Browser (http://localhost:7474)
```

### Indexing

```bash
# Incremental — adds/updates changed files without touching the rest
python -m doxygen_index.cli project . --format neo4j --neo4j-password ticketing-system

# Full rebuild (clears everything first — use when the graph gets out of sync)
python -m doxygen_index.cli project . --format neo4j --clear --neo4j-password ticketing-system
```

### Dashboard

```bash
ticketing-dashboard                              # Default: current directory, port 8082
ticketing-dashboard --port 9000                  # Custom port
ticketing-dashboard --reload --show              # Dev mode with auto-reload + browser
ticketing-dashboard --project-dir ../other-repo  # Point at a different project
ticketing-dashboard --no-start                   # Skip container management (external Neo4j)
```

## Configuration

### `.doxygen-index.toml`

```toml
[project]
name = "ticketing_system"
language = "python"
input_paths = ["backend"]
exclude_patterns = ".venv __pycache__ .pytest_cache .ruff_cache"

[codegraph-db]
password = "ticketing-system"
```

The `[codegraph-db]` section controls the Neo4j container:
| Key | Default | Description |
|---|---|---|
| `password` | `codegraph` | Neo4j `neo4j` user password |
| `image` | `neo4j:5-community` | Docker image |
| `bolt_port` | `7687` | Host port for Bolt protocol |
| `http_port` | `7474` | Host port for HTTP/Browser |

### `.env`

Auto-generated by `codegraph-db start`. Managed block between `# >>> codegraph-db >>>` and `# <<< codegraph-db <<<`:

```
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=ticketing-system
```

The dashboard reads this on startup via `sync_neo4j_config()`.

## Development

### Re-index after code changes

After modifying backend code, re-index incrementally so the codegraph picks up changes without disturbing existing data:

```bash
python -m doxygen_index.cli project . --format neo4j --neo4j-password ticketing-system
```

If the graph gets out of sync (e.g. after renaming or deleting many files), do a full rebuild:

```bash
python -m doxygen_index.cli project . --format neo4j --clear --neo4j-password ticketing-system
```

### Run with auto-reload

```bash
ticketing-dashboard --reload --show
```

### Explore the graph directly

```bash
# Cypher shell
codegraph-db shell

# Neo4j Browser
codegraph-db browser
```

Login: `neo4j` / `ticketing-system`
